---
name: Arabic dubbing (TTS) architecture
description: How optional Arabic voice dubbing is wired across api-server and the youtube-arabic web app
---

Optional, opt-in (default OFF) Arabic voice dubbing for the YouTube-Arabic app.

- TTS provider: ElevenLabs via raw `fetch` in `artifacts/api-server/src/routes/tts.ts` (NOT a shared lib, NOT the Replit AI integration). Header `xi-api-key`, model `eleven_multilingual_v2`, endpoint `/v1/text-to-speech/{voiceId}?output_format=mp3_44100_128`. Requires `ELEVENLABS_API_KEY` secret.
- **Provider choice is the user's call, not a fixed requirement.** The route was implemented against both ElevenLabs and OpenAI at different points; keep the swap localized to `tts.ts` (endpoint/header/model/error-mapping) plus the shared voice allowlist. **Why:** OpenAI quota/billing state can block dubbing even when the code is correct, so the provider may need switching without touching client sync logic.
- **Why raw fetch, not OpenAPI codegen:** the existing `/api/whisper` route already deviates from the contract-first pattern (raw fetch, manual validation), and the dubbing client needs to call TTS in batched loops with progress — a generated React Query mutation hook is a poor fit. Server does manual validation like `translate.ts`.
- Voice IDs are an **allowlist** on the server (`ALLOWED_VOICES`) and must stay in sync with `DUBBING_VOICES` in `artifacts/youtube-arabic/src/lib/storage.ts`. Premade ElevenLabs voices: Rachel/Bella (أنثى), Adam/Antoni (ذكر) — these work on any account.
- Disk cache mirrors `translate.ts`: `cache/tts/<sha1(model|voice|text)>.mp3` (gitignored via `artifacts/api-server/.gitignore`). Empty text → clip `audio: null` (no API call, no charge).
- **Fatal errors:** 401 → Arabic "مفتاح … غير صالح"; 429 → Arabic "تم تجاوز حد … لا يمكن توليد الصوت". A 429 is an account/billing state, not a code bug. First fatal stops further API calls for the batch.
- Client controller: `src/lib/use-dubbing.ts`. YouTube mode = approximate sync (100ms poll + `player.mute()`); Whisper mode = real sync (`<video>` `timeupdate` + `video.muted`). Per-segment HTMLAudio with playbackRate clamped 0.8–2.2 to fit segment duration. `syncTo()` also re-aligns `audio.currentTime` on within-segment seeks (drift > 0.35s).
- **Default on/off preference:** `settings.dubbingEnabled` (default false) persisted in storage. When on, both players auto-activate dubbing once per video via an `autoDubRef` guard (reset on videoId/file change). Default-off means zero behavior change for existing users.
- **Autoplay unlock:** `activate()` plays a silent WAV synchronously inside the toggle click gesture before any await, else browsers block later programmatic `audio.play()`. Auto-activate (no gesture) may stay blocked until the user presses play — acceptable.
- **Voice preview reuses `/api/tts`:** the settings voice picker previews each voice by POSTing a single fixed-text Arabic sample to the same `/tts` route (`use-voice-preview.ts`). No dedicated endpoint — the route's `sha1(model|voice|text)` disk cache makes repeated previews free since the sample text is constant per voice. **Why:** avoids re-billing and keeps the allowlist single-sourced.
- **Dub audio download (`POST /api/tts/download`):** stitches the *already-cached* per-segment clips into one timed MP3 via ffmpeg (`adelay` per clip + `amix=normalize=0`), then streams `audio/mpeg`. Does NOT synthesize/charge — only reads cache; 409 if nothing cached. Clips laid on a cursor timeline (`delay = max(seg.start, prevEnd)`) so overlapping captions don't double voices. Mirror the `/tts` cache key exactly (`slice(MAX_TEXT_LEN).trim()` then `sha1(model|voice|text)`) or every clip misses. Cache dir is resolved from the api-server process cwd = `artifacts/api-server/cache/tts`, NOT workspace root. ffmpeg/ffprobe are available in the Nix env. Client `useDubbing().downloadAudio()` fetch→blob→anchor; button shown only when `dub.status==='ready'`.
- **mp3 duration gotcha:** `ffprobe format=duration` on an mp3 is a *header estimate* and can be ~0.5s off; verify real length with a full decode (`ffmpeg -i x.mp3 -f null -`).
- **Lifecycle gotcha:** `activate()` builds Blob URLs/Audio per chunk. Cancellation paths (`token !== tokenRef.current` early returns) AND failures must release them — handled via a `committed` flag + `finally { if(!committed) disposeClips(clips) }`. `releaseClips` only covers the committed `clipsRef`.
